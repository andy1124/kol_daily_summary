三句話總結

  • AI模型已是Commodity，沒有護城河：從Deepseek到Timmie K2.5再到Seedance，中國開源模型快速迭代，用戶對模型毫無忠誠度，只要效能更好立刻轉換；雲端大模型是商品，Vertical模型才有差異化空間。
  • CXL是HBM下一層的記憶體擴充方案，非替代品：Google可能用CXL做長文本Inference的記憶體容量擴展，但速度仍慢於HBM，兩者形成記憶體階層而非互相取代；拔掉HBM的可能性「低到不可思議」。
  • 美光HBM4落後傳聞影響有限：美光HBM4 Base Die設計問題導致可能不是NVIDIA Vera Rubin平台首批供應商，但整體HBM嚴重短缺，只要通過認證仍可全量出貨；美光財報長漲10%反映市場認為影響短暫。

投資觀點

  • AI支出仍為市場最大驅動力：CSP（Google、Microsoft、Meta、Amazon）資本支出年增率均達50-70%以上，Google甚至發行100年債券鎖定低利率，「停止投資」與「增加投資」都讓市場擔心，但整體方向是持續擴張。
  • 缺貨循環已蔓延至整個半導體供應鏈：設備商法說暗示新產能要2027年才能出來，2026年幾乎整年都缺（設備、晶圓、封裝、記憶體），是自2021年以來最完整的供需緊繃循環。

HBM vs CXL 技術分析

CXL的定位

  • 歷史：CXL聯盟2019年成立，1.0→2.0→3.1演進；過去卡關在Intel/AMD CPU不支援協定，現在3.1版本已支援GPU。
  • 用途：將遠端記憶體透過高速互聯（Switch/網通）連接至GPU，形成記憶體階層：HBM（最快）→CXL（第二層）→Local SSD→遠端SSD。
  • 優點：突破HBM物理空間限制，提供更大容量，適合長文本Inference或影片推理的大Context需求。
  • 缺點：速度仍慢於HBM，是補充而非替代。
  • 手機應用：CXL不適合手機，但NAM Flash（本地SSD）概念可在手機端實現類似的模型容量擴充。

HBM4供應商格局

美光落後傳聞

  • 問題：美光HBM4 Base Die設計難題，Semi Analysis預估份額降至接近0；Samsung今年2月開始量產HBM4，供應Vera Rubin平台，與SK Hynix並列。
  • 影響有限的三個理由：①美光未通過認證只是時間問題，之後仍可供貨；②即使沒有NVIDIA訂單，ASIC自製晶片客戶仍需大量HBM4；③HBM整體嚴重短缺，通過認證後有多少買多少。
  • 美光財報漲10%：財務長出面澄清「已提早一年完成量產，HBM4不實報導」，市場買帳；Vertiv同日因法說表現優異（Backlog Ratio約2.9）大漲帶動板塊。

農曆年後市場展望

缺貨循環2026年主題

  • 結構：AI帶動所有半導體上中下游全面缺貨，是2021年以來最完整的供需循環；且這次買方是CSP（不計成本），拉動時間與幅度比消費性驅動更穩定。
  • 觀察指標：從晶圓出發往上下游追蹤供需Matching程度——晶圓廠從缺開始→設備→封裝，以2021年為歷史參照，邊際變化最大的環節往往是先行指標。
  • 設備商暗示：法說稱新產能2027年才出來，意味2026年整年供不應求機率高；Double Booking在誰身上是關鍵問題。
