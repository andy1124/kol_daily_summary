# 三句話總結

- **Google TPU代數正式釐清**：Ionwood = V7（供應鏈稱V6P），下一代V8P/V8E對應Broadcom全晶片/MTK IO；Marvell切入TPU傳言可信度低，需等V10以後。
- **EMIB封裝可搭配HBM**：澄清市場誤解，Intel EMIB-T有TSV直接支援HBM，伺服器AI晶片使用EMIB完全可行，Intel已拿到相關訂單。
- **軟體股組合策略**：建議拉多支軟體股組合（Confluent/Palo Alto/Cloudflare等）、長期持有，核心看「降本增效能否反映在營業利益」；軟體股在AI應用爆發時將成重要布局標的。

# 投資觀點

- **基本面隔日換/隔週換是大忌**：AI模型競爭白熱化，Google一天被追捧、OpenAI出新版又被看空，基本面不會短期內巨變，避免跟著市場情緒換股。
- **買最強股 vs 拉回才買**：主委偏好稍微拉回（而非追最右上角）時買進，例如最近接交換器大廠在拉回後才進，事後上修報告感覺舒適。
- **組合式持倉降低換股衝動**：軟體組合把個別判斷誤差分散，總體報酬率仍亮眼；避免因某支不動就割掉，到最後可能錯過被收購或爆發的機會。
- **長期持有的代價**：組合中有跑輸的持股（Confluent），但最後被IBM收購退出；印證「你沒走到最後不知道結局」，長線組合整體優於頻繁切換。

# Intel EMIB 封裝與 HBM 澄清

- **市場錯誤認知**：有人認為EMIB不能搭配HBM，主委直接引用Intel官網說明EMIB-T（含TSV孔）可以使用HBM。
- Intel在EMIB這邊已拿到AI伺服器晶片訂單，驗證此技術路線可行。

# Google TPU 供應鏈深度解析

## 代數命名對照表

| 供應鏈稱呼 | Google 官方名稱 | 狀態 |
|---|---|---|
| V6P | V7（Ionwood） | 已推出 |
| V7P / V7E | V8P / V8E | V8E（MTK 20～25K wafer，On Track） |
| V8 | V9 | Ongoing |
| V9 | V10 | 早期規劃 |

- V8E：MTK負責IO晶片，博通負責主晶片（V8P）；2026年V8E on track。
- Marvell切入TPU傳言：可信度低，即使有合作也不是這一顆TPU晶片，可能是V10以後、更周邊的方案；市場可能借OpenAI新版出來時用此傳言打壓Google供應鏈股，屆時視為買點。

# 先進封裝（CoWoS / OSAT）

- 台積電CoWoS擴產仍是瓶頸，多餘需求流向獨立OSAT廠；OSAT廠擴張帶動相關設備股與廠務股需求。
- 主委觀察OSAT/Foundry擴廠的「滾動式修正」：建廠料的缺口是驗證邏輯的交叉比對，Broadcom自己說封裝不夠，等於第三方確認了OSAT需求真實存在。

---

# 先進PCB材料（PTFE）

- 聖鴻（ShinKong）近期有意採用PTFE材料作為背板，原本設計是採用台光電同波機板（搭配Q布）。
- 聖鴻可能因為不想用與自己有競爭關係的廠商的料，轉向台系PTFE業者；這類「解凍原本冷凍的料」是產業動態的常見現象。
- 前述PTFE業者（美國Rogers、台系廠商）若驗過可望受益。

# 光通訊佈局

- Google OCS（光學交換器）採全光方案（無光電轉換），帶動Lumentum、Coherent關注。
- 主委美股光通組合：Lumentum（Google直接採用）+ Coherent（MEMS/hybrid方案，有機會跟進Google路線）。

# 軟體股心得

- **組合配置**：Confluent + Palo Alto + Cloudflare為核心，試布HubSpot、Salesforce，佔比較低。
- **Confluent 被IBM收購**：以20～30%利潤出場；主委形容是「組合裡的軟體之恥」，但最後仍是正向出場，印證長期持有的終局論。
- **未來方向**：等AI應用爆發後，軟體股的「降本增效能否反映在營業利益」將是核心選股標準，可能以巨頭（如Google）搭配小型純軟體股組合。
