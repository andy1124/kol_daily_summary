# 三句話總結

- **Harness Engineering 架構確立**：Anthropic 提出「規劃者＋執行者＋審查者」三角 AI 架構，Token 使用量大增但產出可用性顯著提升；對模型廠商（更多 Token 消耗）與設備廠（算力需求增）均是正向。
- **ASML 法說上修確認 AI CapEx 週期**：全年指引從 340-390 億升至 360-400 億，非中國需求強勁；三奈米製程產能緊張被多方驗證，AI ASIC 採購持續搶佔。
- **MLCC 被動元件漲價循環啟動**：春田電、太陽誘電相繼宣布 5 月漲價，擴及 MLCC、電感、鋁電解電容；AI Server 搶走高規格產能，消費性與工業缺貨效應啟動，形態同 2018-19 年循環。

# 投資觀點

- **Token 消耗量指數爆增是設備需求的最強確認**：Harness Engineering 架構（3 個 AI 角色）相較傳統 2 個角色，Token 消耗量從 20 分鐘暴增至 3 小時；這對 ASML、NVIDIA 等設備/算力廠商是毫無疑問的長期利多。

- **ASML 進入前段循環最強驗證**：廠房蓋好 → 設備採購 → 耗材 → 成品；ASML 法說上修代表整個半導體前段的循環已在第二段（設備採購）高峰期。

- **被動元件跟上漲價**：AI Server 每台使用量達 40-50 萬顆 MLCC 以上，高規格產能被搶占 → 消費性/工業供需吃緊 → 漲價；台廠春田電等 PE 較低者有補漲空間，但需留意替代品競爭。

- **傳統漲價蛛網理論仍然適用**：AI 推動的供需循環與過去記憶體、被動元件模式結構相同，只是觸發點不同；當交期開始縮短，需開始注意高點風險。

# 主題分析區塊

## Harness Engineering（Anthropic 提出）

- **三角 AI 架構**：①規劃者（Planner）制定方案 → ②執行者（Actor）執行 → ③審查者（Reviewer）驗證；解決 AI 自我 Review 過度自信的問題（類比考試後自我修改沒效果）
- **Token 消耗量大增**：從單次提交 20 分鐘，變成規劃＋執行＋審查循環，可能拉長至 3 小時；但最終可用性（不需要人工反覆修改）顯著提升
- **Hermes 開源專案**：基於 Harness Engineering 概念實作，Demo 顯示可用性較傳統 Agent 高；對模型廠商（更多 Token）、算力廠商（更多 GPU）均是利多

## ASML 法說重點

- **全年指引上修**：340-390 億 → 360-400 億美元；毛利率 51-53%（略 miss 但展望好）
- **中國出口限制影響**：目前中國佔銷售比 20%，已回歸 ASML 所稱「長期平均水準」；路透報導若管制再收緊，可能再下修中國比重
- **三奈米產能最緊張**：AI ASIC 客戶大量採用台積電三奈米；Dylan Patel 事前爆料被驗證，EUV 需求確認超強

## MLCC 被動元件漲價循環

- **漲價時程**：春田電 4 月 1 日率先宣布磁性及 EMI 元件漲價；太陽誘電 5 月跟進，擴及 MLCC 電容、電感、鋁電解電容
- **根本原因**：AI Server 每台 MLCC 用量達 40-50 萬顆（HVDC 規格比例提升）；村田、三星電機高階產能被搶占，消費性/工業缺料
- **循環類比 2018-19 年**：當時春田電移產能至車用，現在移至 AI 用；模式相同，只是觸發點從車用換成 AI
- **注意事項**：各品類間（MLCC、固態電容、鋁電容）仍有部分替代性，需分辨哪類最不可替代；交期縮短時為潛在高點訊號

# 操作策略

## 被動元件介入邏輯

- **PE 低者優先**：目前電子零組件整體 PE 相對高，傳產類（被動元件、食品）PE 仍低，安全邊際較大
- **消費性料件是漲價循環中的受益者**：漲價週期中，消費性規格廠商被動受惠（客戶不得不吃漲價），ASP 上升但量下滑，需追蹤毛利率而非純看 ASP
