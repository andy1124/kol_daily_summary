# 財經 Podcast 逐字稿摘要 Skill

## 第一步：載入替換字典

**開始處理前，請先用 Read 工具讀取 `C:\code\kol_daily_summary\dictionary.json`。**

這個字典記錄了已確認的 Whisper 辨識錯誤（格式：`"錯誤詞": "正確詞"`）。在整理摘要時：
- 遇到字典中的錯誤詞，**一律使用正確詞**輸出，不要照抄原文
- 金句區塊例外：若原文本身含有錯誤詞，請改用正確詞後再引用

若字典檔不存在或讀取失敗，跳過此步驟，直接繼續。

---

你的任務是將財經/股票投資 Podcast 的逐字稿，轉換成結構化的**繁體中文** Markdown 筆記。

輸入是 Whisper 原始轉錄，帶有分鐘時間戳（如 `## 00:00`），語言口語化，台灣腔中文混雜英文術語，且常包含與投資無關的個人話題（健身、家庭、廣告等）。你的工作是萃取有投資價值的內容，過濾個人/生活話題，重新組織成高密度的專業筆記。

---

## 過濾原則

**包含**（投資相關）：
- 產業趨勢、公司分析、法說會重點
- 市場觀察、總體經濟、地緣政治對市場影響
- 操作策略、部位管理、槓桿思考
- 財務指標（EPS、毛利率、TAM、市佔率等）
- 投資心態與哲學
- QA 中的財務/投資問題

**排除**（即使占大量篇幅）：
- 廣告贊助段落
- 個人健康、健身、飲食話題
- 家庭瑣事、孩子成長故事
- 與投資無關的 QA（感情、人際、節目互動）
- 閒聊開場白

## 輸出格式

嚴格依照以下順序和格式輸出 Markdown。**不要加入格式中沒有的額外段落**。

---

### 1. 提及公司（必填）

列出逐字稿中出現的所有公司，依市場分類。格式：

```markdown
# 提及公司

- 美股：公司A、公司B、公司C
- 台股：公司D、公司E
- 其他：公司F（市場/國家）
```

- 同一公司只列一次，使用最常見的名稱（可混用中英文，如「聯發科」「NVIDIA」）
- 若某市場無提及公司則省略該行
- 不需要加連結或股票代號（除非逐字稿明確提及）

---

### 2. 三句話總結（必填，剛好三句）

用**恰好三個** bullet points，格式為「**主題詞**：一句話說明」，點出本集最核心的三條投資主線：

```markdown
# 三句話總結

- **主題一**：本集關於此主題的核心發展與意義。
- **主題二**：本集關於此主題的核心發展與意義。
- **主題三**：本集關於此主題的核心發展與意義。
```

選擇最具投資意義的三個主題。主題詞要具體（如「被動元件缺貨」「AMD 法說上修」），不要用泛泛詞語（如「市場行情」）。

---

### 3. 投資觀點（必填）

整理主持人對整體市場、操作策略、產業配置的**一般性投資觀點**，每點傳遞一個明確信息：

```markdown
# 投資觀點

- **觀點標題**：具體說明。若有延伸細節，可用子 bullet 補充。
  * 子細節一
  * 子細節二
- **觀點標題二**：說明。
```

---

### 4. 主題分析區塊（動態，1～5 個）

根據逐字稿中討論的具體話題，自訂主題區塊。每個區塊對應一個明確投資話題：

```markdown
# [主題名稱]

## [子主題（若有）]

- **要點標題**：具體說明，保留數字、比率、時間節點等細節。
  * 補充細節
- **要點標題二**：說明。
```

**常見主題類型**（根據實際內容決定標題）：
- 特定產業分析（如「被動元件供需」「AI 運算硬體」）
- 特定事件（如「金管會政策」「AMD 法說會」「市場壓力測試」）
- 特定公司（如「聯發科估值」「台積電資金行情」）
- 宏觀/地緣政治分析

**重要**：標題應反映實際討論內容，不要套用固定標題。保留具體數字。

---

### 5. 操作策略（若有明確討論則包含，否則省略）

若逐字稿討論了具體的交易操作、部位管理、槓桿策略：

```markdown
# 操作策略

## [策略子主題]

- **策略要點**：說明。
```

若本集無明確操作策略討論，**省略此區塊**。

---

### 6. QA（若有 QA 環節且包含投資相關問答則包含，否則省略）

只整理**投資/財務相關**的 QA 問答，跳過個人感情、人生建議等無關問答：

```markdown
# QA

## [問題主題標題]

- **情境**：一句話說明問題背景。
- **重點回答**：整理主持人回答的核心觀點。
```

若本集 QA 全部為非投資內容，**省略此區塊**。

---

### 7. 節目金句（必填，5～15 句）

從逐字稿中選取最具代表性的**原話**（投資哲學、市場洞察、人生智慧皆可），以引號格式列出：

```markdown
# 節目金句

- 「金句原文，與逐字稿一字不差」
- 「另一句金句」
```

**重要**：
- 必須是主持人說話的**原文**，不可改寫或摘要
- 可以是投資哲學，也可以是人生觀點、犀利評論
- 修剪開頭的贅詞（如以「就是、反正、然後」起頭的句子），但保留說話的感覺
- 避免選太長的段落；超過 80 字的句子請截取最核心的部分

---

### 8. 疑似辨識錯誤（必填）

在整理過程中，若發現逐字稿裡有**不合語境、可能是 Whisper 聽錯的詞**（公司名稱怪異、術語拼法奇特、中文詞組不合理等），請列出你的懷疑：

```markdown
# 疑似辨識錯誤

- 「原文中的詞」→ 建議：「可能的正確詞」（說明原因，如：不存在的公司名、發音相近）
- 「另一個詞」→ 建議：「修正」（說明）
```

**重要規則**：
- 只列出你有一定把握的錯誤，不要亂猜
- 已在 dictionary.json 中有對應修正的詞**不需要**重複列出
- 若本集逐字稿沒有發現疑似錯誤，寫：

```markdown
# 疑似辨識錯誤

無
```

- 這個區塊是給 `detect_typos.py` 自動解析用的，請嚴格遵守格式

---

## 寫作風格

- **精煉不冗長**：每個 bullet 傳遞一個明確信息，刪去「其實」「基本上」等語氣詞
- **數字優先**：保留具體數字、比率、時間節點（如「2027 年 Server CPU TAM 120B」「持股上限 25%」）
- **術語保留**：EPS、ROE、ASIC、GPU、TAM、MLCC、CPO、SaaS 等英文術語直接保留，不翻譯
- **繁體中文**：全文使用繁體中文，英文術語可保留英文
- **中性整理**：整理主持人觀點即可，不加入個人評論

---

## 輸入結構說明

逐字稿通常包含以下幾段，處理方式各異：

| 段落 | 特徵 | 處理方式 |
|------|------|----------|
| 廣告（開頭幾分鐘） | 提到贊助商、優惠碼 | 完全略過 |
| 個人生活分享 | 健身、飲食、家庭故事 | 完全略過 |
| 市場分析（核心） | 公司名稱、財務數字、產業詞 | 詳細整理 |
| QA 環節 | `# QA` 標記後開始 | 只整理投資問題 |
| 閒聊收尾 | 「這集先這樣，掰」 | 略過 |

一個 50 分鐘的節目，通常只有 20～30 分鐘是真正的投資內容。請大膽過濾，輸出的筆記長度約是逐字稿的 15～25%。
